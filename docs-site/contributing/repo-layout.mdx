Kassandra is a monorepo spanning two build systems: a **Rust Cargo workspace** (the on-chain program + the AI runner) and a **pnpm workspace** (the TypeScript SDK + the web app). Documentation and fixture scripts round it out.

## Top-level directories

| Path | What it is | Build system |
| --- | --- | --- |
| `programs/kassandra` | The oracle on-chain program (`kassandra-program`), a Pinocchio-based Solana program. | Cargo (`cargo build-sbf`) |
| `programs/kassandra-market` | The **prediction-market** on-chain program (`kassandra-market-program`) — a Pinocchio wrapper over MetaDAO v0.4 vault + amm, resolved by the oracle. | Cargo (`cargo build-sbf`) |
| `runner` | The AI runner (`kassandra-runner`), a Rust binary/lib crate. | Cargo |
| `sdk` | The oracle TypeScript SDK (`@kassandra/sdk`). | pnpm |
| `sdk-market` | The market TypeScript SDK (`@kassandra-market/sdk`). | pnpm |
| `sdk-rs` | The oracle Rust SDK (`kassandra-sdk`) — builders/PDAs used by the indexer + tests. | Cargo |
| `sdk-rs-market` | The market Rust SDK (`kassandra-market-sdk`) — a solana-sdk v2 "island". | Cargo |
| `indexer` | The single Carbon indexer (`kassandra-indexer`): two pipelines → one Postgres, serving both programs' read + gateway APIs. | Cargo |
| `app` | The single Vite + React dApp — both `/oracles` and `/markets`. | pnpm |
| `docs/` | Design docs and dated implementation plans (`docs/plans/`) for both programs. | — |
| `docs-site/` | This Mintlify documentation site (config `docs.json`, MDX content). | Mintlify / npm |
| `scripts/` | Bash fixture fetchers that dump MetaDAO program binaries for hermetic LiteSVM CPI tests. | — |

## The two workspaces

The Cargo workspace (`Cargo.toml`) uses `resolver = "2"` with members `programs/kassandra`, `programs/kassandra-market`, `runner`, `sdk-rs`, `sdk-rs-market`, and `indexer`. The oracle stack rides the granular **solana v3** client crates; the market program's tests + `sdk-rs-market` are a **solana-sdk v2 island** (pinned directly, not via the workspace), so both majors coexist without a cross-crate type split. The shipped indexer stays v3-clean.

The pnpm workspace (`pnpm-workspace.yaml`) contains:

```yaml
packages:
  - "sdk"
  - "sdk-market"
  - "app"
  - "docs-site"
```

The app consumes **both** SDKs via `workspace:*` and needs their `dist/` built first (`pnpm --filter ./sdk build && pnpm --filter ./sdk-market build`). The Rust crates (`programs/*`, `runner`, `sdk-rs*`, `indexer`) are **not** JS/TS workspace members.

<Note>
  Filter tip: the SDK package is named `@kassandra/sdk`, so `pnpm --filter sdk` does **not**
  match it — use the path form `pnpm --filter ./sdk` (and `./sdk-market`).
</Note>

## Fixture scripts

The two scripts in `scripts/` fetch external MetaDAO program binaries so the CPI tests are hermetic:

- `scripts/fetch-metadao.sh` — dumps the MetaDAO **v0.4** stack used by the challenge markets: the conditional-vault program `VLTX1ishMBbcX3rdBWGssxawAo1Q2X2qxYFYqiGodVg` and the AMM program `AMMyu265tkBpRW21iGQxKGLaves3gKm2JcMUqfXNSpqD`.
- `scripts/fetch-metadao-v06.sh` — additively dumps the **v0.6** futarchy governance stack plus **Meteora DAMM v2** (the `kass_price` governance stack, not the challenge market): futarchy `FUTARELBfJfQ8RDGhg1wdhddq1odMAJUePHFuBYfUxKq`, the same reused conditional-vault, Meteora `cpamdpZCGKUy5JxQXB4dcpGPiikHawvSWAd6mEn1sGG`, and Squads v4 `SQDS4ep65T869zMMBKyuUq6aD6EgTu8psMjkvj52pCf`.
