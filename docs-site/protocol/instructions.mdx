title: "Instructions"
description: "Complete reference for all 24 Kassandra instructions (Ix 0вЂ“23): accounts, payloads, phase gates, and errors."
---

The program exposes **24 instructions**. The first byte of `instruction_data` is the discriminant
(the `Ix` value); the remaining bytes are the payload, parsed inside each processor
(`programs/kassandra/src/instruction.rs`, `processor/mod.rs`). Discriminants are a stable contract вЂ”
never renumbered, only appended.

<Info>
**Reading the account lists.** Slot numbers are the processor's destructure order. "signer" is
enforced on-chain by `assert_signer`; "writable" reflects what the processor creates/mutates (the
runtime enforces the actual meta). A trailing `..` means extra accounts are ignored. Payload sizes are
exact unless noted. Full byte-level payloads are in the [SDK builders](/sdk/instruction-builders);
phase semantics are in [Phases](/protocol/phases).
</Info>

## Init, config & governance

<AccordionGroup>
  <Accordion title="Ix 9 В· InitProtocol вЂ” create the singleton" icon="wrench">
    One-time creation of the `[b"protocol"]` singleton (`processor/init_protocol.rs`). Records the
    admin and canonical KASS/USDC mints, defaults all governable params to the config consts (fee EMA
    zeroed), and uses create-or-adopt to tolerate a pre-funded PDA.

    **Accounts:** `0` protocol PDA (writable, `[b"protocol"]`, self-signs) В· `1` admin (signer,
    writable) В· `2` kass_mint (SPL) В· `3` usdc_mint (SPL) В· `4` system program.

    **Payload:** none (trailing bytes ignored).

    **Errors:** re-init в†’ `AlreadyInitialized`; non-SPL mints в†’ `InvalidAccount`.
  </Accordion>

  <Accordion title="Ix 10 В· CreateOracle вЂ” stand up an oracle" icon="star">
    Stands up a new oracle in `Phase::Proposal` with a future deadline, a program-controlled KASS
    stake vault (`[b"vault", oracle]`, authority = oracle PDA), a burned EMA creation fee, and
    optional emission minted into the vault (`processor/create_oracle.rs`). Snapshots all governable
    params from the Protocol onto the Oracle.

    **Accounts:** `0` protocol (writable) В· `1` oracle PDA (writable, uninit; `[b"oracle", nonce_le]`)
    В· `2` stake_vault PDA (writable, uninit; `[b"vault", oracle]`) В· `3` creator (signer, writable) В·
    `4` kass_mint (writable) В· `5` usdc_mint В· `6` token program В· `7` system program В· `8`
    creator_kass_token (writable, fee burn source) В· `9` mint_authority PDA (`[b"mint_authority"]`).

    **Payload (25 bytes):** `nonce: u64 LE` ++ `options_count: u8` ++ `deadline: i64 LE` ++
    `twap_window: i64 LE`. The question and option labels are not in this payload вЂ” they are written
    on-chain by `WriteOracleMeta` (Ix 23), appended to the same transaction.

    **Errors:** `options_count < 2` в†’ `InvalidOptionsCount`; `deadline < now` в†’ `InvalidDeadline`;
    `twap_window <= 0` в†’ `InvalidInstructionData`; duplicate PDA / bad burn source в†’ `InvalidAccount`;
    emission mint-authority mismatch в†’ `BadMintAuthority`.
  </Accordion>

  <Accordion title="Ix 23 В· WriteOracleMeta вЂ” write on-chain metadata" icon="file-pen">
    Writes the oracle's on-chain metadata into a companion `oracle_meta` PDA
    (`processor/write_oracle_meta.rs`), appended to the **same transaction** as `create_oracle` so the
    metadata lands atomically with the oracle. Stores the plaintext subject (the question), the option
    labels, and a `uri` + `uri_hash` binding the extended off-chain interpretation JSON.

    **Accounts:** `0` creator (signer, writable) В· `1` oracle (read-only) В· `2` oracle_meta PDA
    (writable, uninit; `[b"oracle_meta", oracle]`) В· `3` system program.

    **Payload (variable):** `subject_len: u16 LE` ++ `subject: [u8; subject_len]` ++ `options_count: u8`
    ++ `options_count Г— (option_len: u16 LE ++ option: [u8; option_len])` ++ `uri_len: u16 LE` ++
    `uri: [u8; uri_len]` ++ `uri_hash: [u8;32]`.
  </Accordion>

  <Accordion title="Ix 13 В· SetGovernance вЂ” adminв†’DAO handoff" icon="landmark">
    One-time adminв†’DAO handoff recording `dao_authority` (a Squads v4 multisig vault PDA) and
    `kass_dao` (a futarchy `Dao`), validated against real accounts (`processor/set_governance.rs`).
    After the handoff only the current `dao_authority` may rotate.

    **Accounts:** `0` protocol PDA (writable) В· `1` authority (signer вЂ” `admin` pre-handoff,
    `dao_authority` post) В· `2` kass_dao (futarchy Dao; == payload `kass_dao`).

    **Payload (в‰Ґ 64 bytes):** `dao_authority: [u8;32]` ++ `kass_dao: [u8;32]` (extra trailing bytes
    ignored).

    **Errors:** payload `< 64` в†’ `InvalidInstructionData`; zero key в†’ `InvalidAccount`; wrong signer
    pre-handoff в†’ `Unauthorized`; wrong signer post-handoff в†’ `GovernanceAlreadySet`; bad futarchy Dao
    в†’ `InvalidFutarchyDao`; `dao_authority != derived Squads vault` в†’ `DaoAuthorityMismatch`.
  </Accordion>

  <Accordion title="Ix 14 В· SetConfig вЂ” overwrite governable params" icon="sliders">
    DAO-gated wholesale overwrite of governable Protocol params (monetary + behavioral + reserved +
    challenge fees), bounds-checked (`processor/set_config.rs`). Does **not** touch existing oracles
    (frozen snapshots) or identity/linkage/accounting fields.

    **Accounts:** `0` protocol PDA (writable) В· `1` dao_authority (signer; == `protocol.dao_authority`).

    **Payload (exactly 176 bytes):** 22 Г— 8-byte LE fields, in order: `emission_num, emission_den,
    total_supply_cap, fee_ema_halflife (i64), fee_per_ema_unit, fee_ema_increment, threshold_num,
    threshold_den, market_threshold_num, market_threshold_den, flip_slash_num, flip_slash_den,
    phase_window (i64), proposal_window (i64), fact_vote_slash_num, fact_vote_slash_den,
    reward_proposer_weight, reward_fact_weight, challenge_fail_usdc_fee_num, challenge_fail_usdc_fee_den,
    challenge_success_kass_fee_num, challenge_success_kass_fee_den`.

    **Errors:** wrong signer в†’ `Unauthorized`; any bad bound (zero denominators, `num > den`
    fractions, non-positive windows/halflife, both reward weights zero, or the joint bound
    `flip_num*fee_den + fee_num*flip_den > flip_den*fee_den`) в†’ `InvalidConfig`. See
    [Constants](/protocol/constants).
  </Accordion>

  <Accordion title="Ix 16 В· KassPrice вЂ” read the KASS/USDC TWAP" icon="magnifying-glass-dollar">
    Read-only governance-anchored KASS/USDC spot TWAP from the futarchy `kass_dao`, returned via
    `set_return_data` (`processor/kass_price.rs`). No state change.

    **Accounts:** `0` protocol PDA В· `1` kass_dao (== `protocol.kass_dao`).

    **Payload:** none used. **Return:** 16-byte LE `u128` TWAP (scaled by `KASS_PRICE_SCALE = 1e12`).
  </Accordion>
</AccordionGroup>

## Proposal lifecycle

<AccordionGroup>
  <Accordion title="Ix 11 В· Propose вЂ” register a proposal" icon="hand">
    Registers a proposal (categorical `option` + KASS `bond`) against an oracle in `Proposal` after
    its deadline; escrows the bond into the stake vault; creates one `Proposer` PDA per (oracle,
    authority); enforces `MAX_PROPOSERS` (`processor/propose.rs`). Sets `claim_option =
    CLAIM_OPTION_NONE`.

    **Accounts:** `0` oracle (writable) В· `1` proposer PDA (writable, uninit;
    `[b"proposer", oracle, authority]`) В· `2` authority (signer, writable) В· `3` authority KASS
    (writable, bond source) В· `4` stake vault (writable; == `oracle.stake_vault`) В· `5` token program
    В· `6` system program.

    **Payload (9 bytes):** `option: u8` ++ `bond: u64 LE`.

    **Errors:** `bond == 0` в†’ `ZeroStake`; wrong phase в†’ `WrongPhase`; `now < deadline` в†’
    `DeadlineNotReached`; closed window with proposers в†’ `ProposalWindowClosed`; over cap в†’
    `TooManyProposers`; `option >= options_count` в†’ `InvalidOptionsCount`; duplicate в†’ `DuplicateProposer`.
  </Accordion>

  <Accordion title="Ix 12 В· FinalizeProposals вЂ” close the proposal window" icon="flag-checkered">
    One-shot close of the proposal window (`processor/finalize_proposals.rs`): if all proposers agree
    в†’ `Resolved` with `resolved_option`; if в‰Ґ2 distinct options в†’ open a dispute (set
    `dispute_bond_total = total_oracle_stake`, advance to `FactProposal`, fresh window). No token CPI.

    **Accounts:** `0` oracle (writable, only account mutated) В· `1..N` the **full** proposer set
    (exactly `proposer_count`, read-only, program-owned, tagged Proposer, `.oracle == oracle`,
    distinct).

    **Payload:** empty.

    **Errors:** wrong phase в†’ `WrongPhase`; window not elapsed в†’ `WindowNotElapsed`;
    `proposer_count == 0` в†’ `NoProposals`; wrong tail length / over cap в†’ `InvalidAccount`.
  </Accordion>

  <Accordion title="Ix 7 В· AdvancePhase вЂ” permissionless freeze" icon="forward">
    Permissionless, time-gated `FactProposal в†’ FactVoting` freeze once the fact-proposal window
    elapsed (`processor/advance_phase.rs`). **No signer required.** Handles only this one edge.

    **Accounts:** `0` oracle (writable). **Payload:** empty.

    **Errors:** wrong phase в†’ `WrongPhase`; window not elapsed в†’ `WindowNotElapsed`.
  </Accordion>
</AccordionGroup>

## Fact phase

<AccordionGroup>
  <Accordion title="Ix 0 В· SubmitFact вЂ” propose a supporting fact" icon="file-lines">
    Proposes a supporting `Fact` during the `FactProposal` window; escrows submitter KASS stake into
    the stake vault (`processor/submit_fact.rs`).

    **Accounts:** `0` oracle (writable) В· `1` fact PDA (writable, uninit;
    `[b"fact", oracle, content_hash]`) В· `2` submitter (signer, writable) В· `3` submitter KASS
    (writable, stake source; mint == `oracle.kass_mint`) В· `4` stake vault (writable;
    == `oracle.stake_vault`) В· `5` token program В· `6` system program.

    **Payload (variable, min 42 bytes):** `content_hash: [u8;32]` ++ `stake: u64 LE` ++
    `uri_len: u16 LE` ++ `uri: [u8; uri_len]`. `uri_len <= 200`; no trailing bytes.

    **Errors:** wrong phase в†’ `WrongPhase`; window closed в†’ `WindowClosed`; `stake == 0` в†’ `ZeroStake`;
    duplicate в†’ `DuplicateFact`.
  </Accordion>

  <Accordion title="Ix 1 В· VoteFact вЂ” vote on a fact" icon="check-to-slot">
    Stake-weighted approve/duplicate vote on a fact during `FactVoting`; one vote per (fact, voter)
    (`processor/vote_fact.rs`).

    **Accounts:** `0` oracle (writable) В· `1` fact (writable; `fact.oracle == oracle`) В· `2`
    fact_vote PDA (writable, uninit; `[b"vote", fact, voter]`) В· `3` voter (signer, writable) В· `4`
    voter KASS (writable, stake source) В· `5` stake vault (writable; == `oracle.stake_vault`) В· `6`
    token program В· `7` system program.

    **Payload (exactly 9 bytes):** `kind: u8` (`VOTE_APPROVE=0` / `VOTE_DUPLICATE=1`) ++ `stake: u64 LE`.

    **Errors:** wrong phase в†’ `WrongPhase`; window closed в†’ `WindowClosed`; `stake == 0` в†’ `ZeroStake`;
    duplicate в†’ `DuplicateVote`; bad `kind` в†’ `InvalidInstructionData`.
  </Accordion>

  <Accordion title="Ix 2 В· FinalizeFacts вЂ” settle the fact round" icon="scale-balanced">
    Incrementally settles the fact-voting round after its window (`processor/finalize_facts.rs`):
    classifies each fact (agreed/duplicate/rejected), accumulating `bond_pool` and
    `total_approved_fact_stake`, advancing to `AiClaim` when all facts are settled; OR the no-facts
    dead-end that slashes proposers and terminates in `InvalidDeadend` (burning `bond_pool +
    reward_emission`). Agreed test: `approve_stake > duplicate_stake && approve_stakeВ·threshold_den >=
    dispute_bond_totalВ·threshold_num` (u128).

    **Accounts:** `0` oracle (writable, signs the dead-end burn) В· `1` kass_mint (writable;
    == `oracle.kass_mint`) В· `2` stake_vault (writable; == `oracle.stake_vault`) В· `3` token program В·
    `4..N` a non-empty tail вЂ” a subset of the oracle's facts (fact path) OR proposers (no-facts path),
    each writable, program-owned, `.oracle == oracle`, distinct.

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE` (re-derives the oracle PDA for the burn
    signer).

    **Errors:** wrong phase в†’ `WrongPhase`; window not elapsed в†’ `WindowNotElapsed`;
    `dispute_bond_total == 0` в†’ `NoDisputeBond`; empty tail в†’ `IncompleteFactSet`; already-settled в†’
    `AlreadySettled`.
  </Accordion>
</AccordionGroup>

## AI claim

<AccordionGroup>
  <Accordion title="Ix 3 В· SubmitAiClaim вЂ” submit a categorical claim" icon="robot">
    A locked-in proposer submits a categorical AI claim + reproducibility metadata; records
    `claim_option` on the proposer (marking `flipped` if it differs from `original_option`). No token
    CPI (`processor/submit_ai_claim.rs`).

    **Accounts:** `0` oracle (writable) В· `1` proposer PDA (writable, the submitter's) В· `2` ai_claim
    PDA (writable, uninit; `[b"claim", oracle, proposer]`) В· `3` authority (signer, writable; ==
    `proposer.authority`) В· `4` system program.

    **Payload (exactly 97 bytes):** `model_id: [u8;32]` ++ `params_hash: [u8;32]` ++ `io_hash: [u8;32]`
    ++ `option: u8`. See [AI runner](/architecture/runner).

    **Errors:** wrong phase в†’ `WrongPhase`; window closed в†’ `WindowClosed`;
    `proposer.oracle != oracle` в†’ `InvalidAccount`; wrong/disqualified authority в†’ `Unauthorized`;
    `option >= options_count` в†’ `InvalidOption`; duplicate в†’ `DuplicateClaim`.
  </Accordion>

  <Accordion title="Ix 8 В· FinalizeAiClaims вЂ” settle the AI-claim round" icon="list-check">
    Incrementally settles the AI-claim round after its window (`processor/finalize_ai_claims.rs`):
    no-show (`claim_option == CLAIM_OPTION_NONE`) в†’ full slash + disqualify + `surviving_count -= 1`;
    flipped в†’ partial slash `bondВ·flip_slash_num/flip_slash_den` (stays surviving); submitted &
    not-flipped в†’ no slash. Advances to `Challenge` when `ai_finalized_count == proposer_count`. No
    token CPI.

    **Accounts:** `0` oracle (writable) В· `1..N` a non-empty subset of proposers (each writable,
    program-owned, `.oracle == oracle`, distinct).

    **Payload:** empty.

    **Errors:** wrong phase в†’ `WrongPhase`; window not elapsed в†’ `WindowNotElapsed`; empty tail в†’
    `IncompleteFactSet`; already ai-finalized в†’ `AlreadySettled`.
  </Accordion>
</AccordionGroup>

## Challenge

<AccordionGroup>
  <Accordion title="Ix 4 В· OpenChallenge вЂ” open a decision market" icon="gavel">
    Opens a MetaDAO binary decision market against an `AiClaim` during `Challenge`
    (`processor/open_challenge.rs`). Verifies that the MetaDAO question/vaults/AMMs bind to this
    oracle, records a `Market` PDA, program-signed-splits the proposer bond into pass/fail conditional
    KASS, sizes and funds a challenger USDC escrow via `kass_price`, sets `ai_claim.challenged = 1`,
    and increments `open_challenge_count`. Escrow sizing: `required_usdc = bond В· twap /
    KASS_PRICE_SCALE` where `twap = kass_price(protocol, kass_dao)`.

    **Accounts (25):** `0` oracle (writable, split authority) В· `1` ai_claim (writable) В· `2`
    proposer (writable, bond source) В· `3` market PDA (writable, uninit; `[b"market", ai_claim]`) В·
    `4` challenger (signer, writable) В· `5` question (MetaDAO, resolver == oracle, `num_outcomes==2`) В·
    `6` kass_vault (writable) В· `7` usdc_vault В· `8` pass_amm В· `9` fail_amm В· `10` stake_vault
    (writable) В· `11` kass_vault_underlying_ata (writable) В· `12` pass_kass_mint (writable) В· `13`
    fail_kass_mint (writable) В· `14` oracle_pass_kass (writable) В· `15` oracle_fail_kass (writable) В·
    `16` conditional_vault program В· `17` token program В· `18` system program В· `19` cv_event_authority
    В· `20` protocol В· `21` kass_dao (== `protocol.kass_dao`) В· `22` usdc_mint (== `oracle.usdc_mint`) В·
    `23` challenger_usdc_src (writable) В· `24` challenger_usdc_vault (writable, uninit;
    `[b"challenge_usdc", market]`, authority = oracle PDA).

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE`.

    **Errors:** wrong phase в†’ `WrongPhase`; window closed в†’ `WindowClosed`; already challenged в†’
    `AlreadyChallenged`; disqualified proposer в†’ `Unauthorized`; MetaDAO binding failures в†’
    `InvalidAccount`; `required_usdc == 0` в†’ `ZeroStake`. See [Composing a market](/challenge/composing).
  </Accordion>

  <Accordion title="Ix 5 В· SettleChallenge вЂ” settle one market" icon="scale-unbalanced">
    Settles ONE challenged claim's market after its TWAP window
    (`processor/settle_challenge.rs`). Hard-binds both AMMs to this market's conditional mint pair,
    reads slot-weighted TWAPs, applies the slash trigger, resolves the MetaDAO question, physically
    redeems the bond KASS back to `stake_vault`, and routes directional fees. Phase stays `Challenge`;
    decrements `open_challenge_count`.

    **Slash trigger:** disqualify iff `pass_twap != 0 && fail_twapВ·market_threshold_den >
    pass_twapВ·(market_threshold_den + market_threshold_num)` (u128). A `pass_twap == 0` always
    survives. See [Settle verdict](/challenge/settle).

    **Accounts (21):** `0` oracle (writable) В· `1` market (writable) В· `2` ai_claim В· `3` proposer
    (writable) В· `4` question (writable) В· `5` pass_amm В· `6` fail_amm В· `7` conditional_vault program
    В· `8` cv_event_authority В· `9` token program В· `10` stake_vault (writable) В· `11` kass_vault
    (writable) В· `12` kass_vault_underlying (writable) В· `13` pass_kass_mint (writable) В· `14`
    fail_kass_mint (writable) В· `15` oracle_pass_kass (writable) В· `16` oracle_fail_kass (writable) В·
    `17` challenger_usdc_vault (writable) В· `18` proposer_usdc (writable) В· `19` challenger_usdc_dest
    (writable) В· `20` challenger_kass (writable).

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE`.

    **Errors:** wrong phase в†’ `WrongPhase`; already settled в†’ `AlreadySettled`;
    `now < market.twap_end` в†’ `TwapWindowOpen`; `pass_amm == fail_amm` в†’ `InvalidAccount`.
  </Accordion>
</AccordionGroup>

## Resolution

<AccordionGroup>
  <Accordion title="Ix 6 В· FinalizeOracle вЂ” final plurality recompute" icon="trophy">
    One-shot final plurality recompute over surviving proposers, driving the oracle terminal
    (`processor/finalize_oracle.rs`). `Winner(opt)` в†’ `Resolved`, stamping
    `total_correct_proposer_stake` and `reward_pool = bond_pool + reward_emission`; `Tie` /
    `NoSurvivors` в†’ `InvalidDeadend`, `resolved_option = CLAIM_OPTION_NONE`, burning
    `reward_emission + bond_pool`.

    **Accounts:** `0` oracle (writable, signs burn) В· `1` kass_mint (writable; == `oracle.kass_mint`)
    В· `2` stake_vault (writable; == `oracle.stake_vault`) В· `3` token program В· `4..N` the **full**
    proposer set (exactly `proposer_count`, read-only, program-owned, tagged Proposer,
    `.oracle == oracle`, distinct).

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE`.

    **Errors:** wrong phase в†’ `WrongPhase`; window not elapsed в†’ `WindowNotElapsed`;
    `open_challenge_count != 0` в†’ `ChallengesOutstanding`; bad tail length / over cap / surviving
    proposer with `CLAIM_OPTION_NONE` / miscounted survivors в†’ `InvalidAccount`.
  </Accordion>

  <Accordion title="Ix 15 В· ResolveDeadend вЂ” stamp a dead-end resolution" icon="rotate-right">
    DAO-gated stamping of a final `resolved_option` on a dead-ended oracle, advancing `InvalidDeadend
    в†’ Resolved` (`processor/resolve_deadend.rs`). No token movement (dead-end funds were already
    burned at finalize).

    **Accounts:** `0` protocol PDA (reads dao_authority) В· `1` oracle (writable) В· `2` dao_authority
    (signer; == `protocol.dao_authority`).

    **Payload (exactly 1 byte):** `option: u8`.

    **Errors:** wrong signer в†’ `Unauthorized`; not `InvalidDeadend` в†’ `WrongPhase`;
    `option >= options_count` в†’ `InvalidOptionsCount`.
  </Accordion>
</AccordionGroup>

## Claim, close & sweep

<AccordionGroup>
  <Accordion title="Ix 17 В· ClaimProposer вЂ” pull a proposer payout" icon="coins">
    Permissionless per-proposer pull-claim after the oracle is terminal; pays the KASS entitlement
    from `stake_vault` and CLOSES the `Proposer` (rent в†’ authority) (`processor/claims.rs`). Idempotent
    by closure. Entitlement: `base = is_disqualified ? 0 : bond в€’ slashed_amount`, plus
    `proposer_reward` only when Resolved + surviving + `claim_option == resolved_option`.

    **Accounts:** `0` oracle В· `1` proposer (writable, CLOSED) В· `2` dest_kass (writable;
    owner == `proposer.authority`) В· `3` stake_vault (writable; == `oracle.stake_vault`) В· `4`
    rent_recipient (writable; == `proposer.authority`) В· `5` token program.

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE`.

    **Errors:** not terminal в†’ `WrongPhase`; bad oracle PDA / bad dest / rent recipient в†’ `InvalidAccount`.
  </Accordion>

  <Accordion title="Ix 18 В· ClaimFact вЂ” pull a fact-submitter payout" icon="coins">
    Permissionless per-fact-submitter pull-claim; CLOSES the `Fact` (rent в†’ `fact.proposer`). Must run
    LAST, after all voters (`processor/claims.rs`). Entitlement: agreed в†’ `stake + fact_reward`
    (reward on Resolved only); duplicate в†’ `stake`; rejected в†’ `0`.

    **Accounts:** `0` oracle В· `1` fact (writable, CLOSED) В· `2` dest_kass (writable;
    owner == `fact.proposer`) В· `3` stake_vault (writable) В· `4` rent_recipient (writable;
    == `fact.proposer`) В· `5` token program.

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE`.

    **Errors:** not terminal в†’ `WrongPhase`; `approve_stake != 0 || duplicate_stake != 0` в†’
    `VotersOutstanding`.
  </Accordion>

  <Accordion title="Ix 19 В· ClaimFactVote вЂ” pull a voter payout" icon="coins">
    Permissionless per-vote pull-claim; decrements the Fact's running voter-stake total, pays KASS,
    and CLOSES the `FactVote` (rent в†’ voter). The Fact is loaded to read disposition but NOT closed
    (`processor/claims.rs`). Entitlement: duplicate vote в†’ `stake`; approve on agreed в†’
    `stake + fact_reward` (Resolved only); approve on duplicate-fact в†’ `stake`; approve on rejected в†’
    `stake в€’ ceil(stakeВ·fact_vote_slash_num/fact_vote_slash_den)`.

    **Accounts:** `0` oracle В· `1` fact_vote (writable, CLOSED) В· `2` fact (writable, total
    decremented) В· `3` dest_kass (writable; owner == `vote.voter`) В· `4` stake_vault (writable) В· `5`
    rent_recipient (writable; == `vote.voter`) В· `6` token program.

    **Payload (exactly 8 bytes):** `oracle_nonce: u64 LE`.

    **Errors:** not terminal в†’ `WrongPhase`.
  </Accordion>

  <Accordion title="Ix 20 В· CloseAiClaim вЂ” reclaim claim rent" icon="box-archive">
    Permissionless rent-reclaim close for one `AiClaim` after terminal; drains rent to
    `ai_claim.authority`; no token movement; order-independent (`processor/close_ai_claim.rs`).

    **Accounts:** `0` oracle (terminal) В· `1` ai_claim (writable, CLOSED) В· `2` rent_recipient
    (writable; == `ai_claim.authority`).
