title: "Accounts"
description: "Complete reference for the Kassandra account structs — full field tables, byte sizes, AccountType tags, and the enums and sentinels."
---

Kassandra stores state in the account types defined in `programs/oracles/src/state.rs`. The
fixed-layout structs are `#[repr(C)]`, `Pod + Zeroable`, fully packed (explicit `_pad` arrays), and
read/written zero-copy via `bytemuck`, each beginning with an 8-byte header: `account_type: u8` at
offset 0 followed by `_pad_hdr: [u8;7]`. The one exception is `oracle_meta`, a variable-length
metadata account with a compact 34-byte header (see below). Byte sizes and offsets are pinned by `programs/oracles/tests/state_layout.rs`.
Throughout, `type Pubkey = [u8;32]`.

## Enums & sentinels

### AccountType (`#[repr(u8)]`)

Stored as the first byte of every account to reject type-confusion. A freshly zeroed `CreateAccount`'d
account carries `Uninitialized` (0) until it is stamped.

| Variant | Value |
| --- | --- |
| `Uninitialized` | 0 |
| `Oracle` | 1 |
| `Proposer` | 2 |
| `Fact` | 3 |
| `FactVote` | 4 |
| `AiClaim` | 5 |
| `Market` | 6 |
| `Protocol` | 7 |
| `OracleMeta` | 8 |

### Phase (`#[repr(u8)]`)

Stored on `Oracle.phase`. Values 0–8; see [Phases](/protocol/phases) for the full transition map.
Pinned: `Created as u8 == 0`, `InvalidDeadend as u8 == 8`, `from_u8(9) == None`.

### Plurality (in-memory only)

The return type of the plurality resolution (`plurality.rs`), never stored: `Winner(u8)` (the
strictly-highest option), `Tie` (≥2 options share the maximum), `NoSurvivors` (empty slice).

### Sentinels (`state.rs`)

- `CLAIM_OPTION_NONE: u8 = 0xFF` — `Proposer.claim_option` "no AI claim yet"; also stamped into
  `Oracle.resolved_option` on `InvalidDeadend`.
- `VOTE_APPROVE: u8 = 0`, `VOTE_DUPLICATE: u8 = 1` — the `FactVote.kind` values.

## Protocol — LEN 368

The program singleton / global config. Created once by `init_protocol`, never re-initialized. Pins the
canonical KASS/USDC mints and carries the creation-fee EMA state plus the governable
monetary/behavioral params (the mutable source snapshotted onto each Oracle). **PDA:** `[b"protocol"]`.

| Field | Type | Offset | Meaning |
| --- | --- | --- | --- |
| `account_type` | u8 | 0 | `AccountType::Protocol` |
| `_pad_hdr` | [u8;7] | 1 | header padding |
| `admin` | Pubkey | 8 | initializer; gates one-time `set_governance` |
| `kass_mint` | Pubkey | 40 | canonical KASS mint |
| `usdc_mint` | Pubkey | 72 | canonical USDC mint |
| `fee_ema` | u64 | 104 | fixed-point activity EMA (scaled by `FEE_EMA_SCALE`); 0 at genesis |
| `last_creation_unix` | i64 | 112 | unix of most recent oracle creation (EMA decay); 0 at genesis |
| `bump` | u8 | 120 | PDA bump |
| `governance_set` | u8 | 121 | 1 once `set_governance` recorded the DAO linkage (one-shot) |
| `_pad` | [u8;6] | 122 | padding |
| `dao_authority` | Pubkey | 128 | Squads v4 multisig vault PDA; gates `set_config` & `resolve_deadend`; zero until set |
| `kass_dao` | Pubkey | 160 | futarchy `Dao`; its embedded spot AMM is the KASS price source; zero until set |
| `emission_num` | u64 | 192 | emission rate numerator |
| `emission_den` | u64 | 200 | emission rate denominator |
| `total_supply_cap` | u64 | 208 | hard cap on circulating KASS supply |
| `fee_ema_halflife` | i64 | 216 | mirror of `FEE_EMA_HALFLIFE_SECS` |
| `fee_per_ema_unit` | u64 | 224 | mirror of `FEE_PER_EMA_UNIT` |
| `fee_ema_increment` | u64 | 232 | mirror of `FEE_EMA_INCREMENT` |
| `threshold_num` | u64 | 240 | fact-quorum supermajority numerator |
| `threshold_den` | u64 | 248 | fact-quorum supermajority denominator |
| `market_threshold_num` | u64 | 256 | slash-trigger margin numerator |
| `market_threshold_den` | u64 | 264 | slash-trigger margin denominator |
| `flip_slash_num` | u64 | 272 | flip-slash fraction numerator |
| `flip_slash_den` | u64 | 280 | flip-slash fraction denominator |
| `phase_window` | i64 | 288 | dispute phase window seconds |
| `proposal_window` | i64 | 296 | proposal-registration window seconds |
| `fact_vote_slash_num` | u64 | 304 | reserved (voter slash) numerator |
| `fact_vote_slash_den` | u64 | 312 | reserved denominator |
| `reward_proposer_weight` | u64 | 320 | reward weight PW |
| `reward_fact_weight` | u64 | 328 | reward weight FW |
| `challenge_fail_usdc_fee_num` | u64 | 336 | USDC fee on failed challenge numerator |
| `challenge_fail_usdc_fee_den` | u64 | 344 | denominator |
| `challenge_success_kass_fee_num` | u64 | 352 | KASS fee on successful challenge numerator |
| `challenge_success_kass_fee_den` | u64 | 360 | denominator |

The governable params are detailed in [Constants](/protocol/constants). Identity/accounting fields
(`admin`, mints, `fee_ema`, `last_creation_unix`, `bump`, `governance_set`, `dao_authority`,
`kass_dao`) are **not** touched by `set_config`.

## Oracle — LEN 360

The top-level dispute account. Its governable params are **snapshotted from `Protocol` at
`create_oracle`** and read by downstream processors from the Oracle, so mid-dispute governance changes
cannot move the goalposts. **PDA:** `[b"oracle", nonce_le]`.

| Field | Type | Offset | Meaning |
| --- | --- | --- | --- |
| `account_type` | u8 | 0 | `AccountType::Oracle` |
| `_pad_hdr` | [u8;7] | 1 | header padding |
| `creator` | Pubkey | 8 | oracle creator |
| `kass_mint` | Pubkey | 40 | KASS mint (must match Protocol) |
| `usdc_mint` | Pubkey | 72 | USDC mint |
| `stake_vault` | Pubkey | 104 | PDA token account holding all KASS bonds/stakes |
| `deadline` | i64 | 136 | unix; proposals rejected before this |
| `phase_ends_at` | i64 | 144 | end of the current window |
| `twap_window` | i64 | 152 | per-oracle TWAP window, seconds |
| `options_count` | u8 | 160 | number of categorical options |
| `phase` | u8 | 161 | `Phase` as u8 |
| `proposer_count` | u16 | 162 | total proposers |
| `surviving_count` | u16 | 164 | proposers not disqualified |
| `fact_count` | u16 | 166 | number of facts |
| `total_oracle_stake` | u64 | 168 | conservation accumulator; == stake_vault balance until a challenge splits a bond |
| `bond_pool` | u64 | 176 | accumulated slashed KASS (base units) |
| `dispute_bond_total` | u64 | 184 | Σ proposer bonds, fixed at dispute start; fact-quorum denominator |
| `settled_count` | u16 | 192 | facts settled so far |
| `ai_finalized_count` | u16 | 194 | proposers ai-finalized so far |
| `bump` | u8 | 196 | PDA bump |
| `resolved_option` | u8 | 197 | final option; valid ONLY when `phase == Resolved`; `0xFF` on `InvalidDeadend` |
| `open_challenge_count` | u16 | 198 | # open (created-but-unsettled) challenge markets; finalize requires 0 |
| `threshold_num` | u64 | 200 | fact-quorum supermajority numerator |
| `threshold_den` | u64 | 208 | fact-quorum supermajority denominator |
| `market_threshold_num` | u64 | 216 | slash-trigger margin numerator (widened to u128 on use) |
| `market_threshold_den` | u64 | 224 | slash-trigger margin denominator |
| `flip_slash_num` | u64 | 232 | flip-slash fraction numerator |
| `flip_slash_den` | u64 | 240 | flip-slash fraction denominator |
| `phase_window` | i64 | 248 | dispute phase window seconds |
| `proposal_window` | i64 | 256 | proposal-registration window seconds |
| `fact_vote_slash_num` | u64 | 264 | reserved (voter slash) numerator |
| `fact_vote_slash_den` | u64 | 272 | reserved denominator |
| `reward_proposer_weight` | u64 | 280 | reward weight PW |
| `reward_fact_weight` | u64 | 288 | reward weight FW |
| `challenge_fail_usdc_fee_num` | u64 | 296 | USDC fee on failed challenge numerator |
| `challenge_fail_usdc_fee_den` | u64 | 304 | denominator |
| `challenge_success_kass_fee_num` | u64 | 312 | KASS fee on successful challenge numerator |
| `challenge_success_kass_fee_den` | u64 | 320 | denominator |
| `total_correct_proposer_stake` | u64 | 328 | Σ bond over surviving proposers whose `claim_option == resolved_option` |
| `total_approved_fact_stake` | u64 | 336 | Σ (`fact.stake` + `fact.approve_stake`) over agreed facts |
| `reward_pool` | u64 | 344 | distributable pool at resolution; `= bond_pool + reward_emission` on Resolved; 0 on InvalidDeadend |
| `reward_emission` | u64 | 352 | KASS minted into stake_vault at create; folded into reward_pool on Resolved, burned on InvalidDeadend |

## OracleMeta — variable

The companion metadata account for one oracle, written by `write_oracle_meta` (`Ix::WriteOracleMeta =
23`) in the same transaction as `create_oracle`. Unlike the fixed-layout structs it is
**variable-length** and length-prefixed: it holds the plaintext subject (the question), the option
labels, and a `uri` + `uri_hash` binding the extended off-chain interpretation JSON. **PDA:**
`[b"oracle_meta", oracle]`.

Byte layout (in order):

| Field | Type | Meaning |
| --- | --- | --- |
| `account_type` | u8 | `AccountType::OracleMeta` (8) |
| `bump` | u8 | PDA bump |
| `oracle` | Pubkey | parent oracle |
| `subject_len` | u16 | length of the subject bytes |
| `subject` | [u8; subject_len] | plaintext question |
| `options_count` | u8 | number of option labels that follow |
| `options` | `options_count × (option_len: u16 ++ option: [u8; option_len])` | length-prefixed option labels |
| `uri_len` | u16 | length of the uri bytes |
| `uri` | [u8; uri_len] | URL of the extended off-chain interpretation JSON |
| `uri_hash` | [u8;32] | `sha256` of that JSON; the runner refuses to resolve unless it matches |

## Proposer — LEN 96

A proposer's commitment within an oracle. **PDA:** `[b"proposer", oracle, authority]`.

| Field | Type | Offset | Meaning |
| --- | --- | --- | --- |
| `account_type` | u8 | 0 | `AccountType::Proposer` |
| `_pad_hdr` | [u8;7] | 1 | header padding |
| `oracle` | Pubkey | 8 | parent oracle |
| `authority` | Pubkey | 40 | human authority |
| `bond` | u64 | 72 | locked KASS |
| `original_option` | u8 | 80 | value at proposal time |
| `claim_option` | u8 | 81 | value after AI claim; `0xFF` = not yet submitted (must init to 0xFF, not zeroed) |
| `disqualified` | u8 | 82 | bool |
| `slashed` | u8 | 83 | bool |
| `flipped` | u8 | 84 | bool: `claim_option != original_option` |
| `bump` | u8 | 85 | PDA bump |
| `ai_finalized` | u8 | 86 | bool: settled by `finalize_ai_claims` |
| `_pad` | [u8;1] | 87 | padding |
| `slashed_amount` | u64 | 88 | KASS slashed from this proposer into `bond_pool` |

## Market — LEN 416

A challenge decision-market binding for one `AiClaim`, created lazily by `open_challenge`. Records the
MetaDAO accounts, the oracle-PDA-owned conditional-KASS destinations, and the challenger's escrowed
USDC. **PDA:** `[b"market", ai_claim]`.

| Field | Type | Offset | Meaning |
| --- | --- | --- | --- |
| `account_type` | u8 | 0 | `AccountType::Market` |
| `_pad_hdr` | [u8;7] | 1 | header padding |
| `oracle` | Pubkey | 8 | parent oracle |
| `ai_claim` | Pubkey | 40 | challenged claim |
| `proposer` | Pubkey | 72 | proposer being challenged |
| `challenger` | Pubkey | 104 | challenger |
| `question` | Pubkey | 136 | MetaDAO binary question (resolver == oracle PDA) |
| `kass_vault` | Pubkey | 168 | MetaDAO conditional vault, underlying == `oracle.kass_mint` |
| `usdc_vault` | Pubkey | 200 | MetaDAO conditional vault, underlying == `oracle.usdc_mint` |
| `pass_amm` | Pubkey | 232 | outcome-0 (pass) AMM |
| `fail_amm` | Pubkey | 264 | outcome-1 (fail) AMM |
| `oracle_pass_kass` | Pubkey | 296 | oracle-PDA-owned conditional-KASS token acct, pass side |
| `oracle_fail_kass` | Pubkey | 328 | oracle-PDA-owned conditional-KASS token acct, fail side |
| `challenger_usdc_vault` | Pubkey | 360 | market-owned USDC escrow (`[b"challenge_usdc", market]`) |
| `twap_end` | i64 | 392 | `now + oracle.twap_window`; settle allowed only after |
| `challenger_usdc` | u64 | 400 | challenger's escrowed USDC = `bond × kass_price` (raw base units) |
| `settled` | u8 | 408 | bool; set by `settle_challenge` |
| `bump` | u8 | 409 | PDA bump |
| `_pad` | [u8;6] | 410 | padding |
