Kassandra's on-chain program is a Pinocchio-based crate (`kassandra-oracles-program`) under `programs/oracles`. This page covers building the SBF artifact and deploying it.

## Toolchain

The workspace pins **stable** Rust (`rust-toolchain.toml`: `channel = "stable"`). The Cargo workspace (`Cargo.toml`) uses `resolver = "2"`, members `programs/oracles` and `runner`, and sets `[profile.release] overflow-checks = true`. Building the SBF artifact additionally requires the Solana SBF toolchain (`cargo build-sbf`).

## Build the SBF program

The `justfile` at the repo root wraps the build in a single target:

```bash
just build
```

which runs, verbatim (`justfile`):

```bash
cargo build-sbf --manifest-path programs/oracles/Cargo.toml
```

This produces the program's `.so` artifact. The runner reuses the same program crate with the `no-entrypoint` feature (`runner/Cargo.toml`), so a program layout change never drifts from the runner's encoding.

<Warning>
The LiteSVM program tests `include_bytes!` the compiled `.so`. Always rebuild
before testing a program change, or you will test a stale artifact. The `just test`
target enforces this — it depends on `build`:

```
test: build
    cargo test -p kassandra-oracles-program
```

Run `just test` (never a bare `cargo test`) when verifying a merged program change.
</Warning>

## Deploy

Deploy the built artifact with the standard Solana CLI:

```bash
solana program deploy target/deploy/kassandra_oracles_program.so
```

Point the CLI at the target cluster with `solana config set --url <rpc>` (or `--url` on the command) and fund the deploying keypair. The program's declared id is `KassVxvXUEPr5apSr2MqiGva4VFtJXyYLLDFS3f83nY` (`sdk/src/constants.ts` `KASSANDRA_PROGRAM_ID`).

<Note>
For the challenge markets, a cluster also needs the MetaDAO v0.4 conditional-vault
and AMM programs present. Localnet/surfpool tests dump these as fixtures via
`scripts/fetch-metadao.sh` (see [Repo layout](/contributing/repo-layout)); on
mainnet-beta they are already deployed.
</Note>
