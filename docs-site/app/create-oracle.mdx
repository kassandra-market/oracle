The route `/oracles/new` renders `app/src/pages/CreateOracle.tsx`, the form that mints a new oracle. It is the first wallet-signed action in the lifecycle. On success it navigates to the new oracle's detail page.

The form wraps the pure action layer at `app/src/data/actions/create.ts`, which builds the `create_oracle` instruction via the SDK's `createOracle` builder (`sdk/src/instructions/lifecycle.ts`, Ix = 10).

## Form fields

<ParamField path="question" type="string">
  The oracle's question in plain text. It is stored **on-chain** (as the subject
  of the companion `oracle_meta` account) so other programs — and the AI runner —
  can read it directly without the indexer.
</ParamField>

<ParamField path="options" type="string[]">
  The categorical option labels. Stored on-chain in `oracle_meta`; their count
  drives `Oracle.options_count`.
</ParamField>

<ParamField path="Advanced (optional)" type="metadata">
  A `promptTemplate` (the AI-runner interpretation, defaulted from the question),
  an optional human `interpretation`, and a `category`. These go into an extended
  JSON hosted off-chain at the oracle's `uri`, bound by a 32-byte `uri_hash` on
  `oracle_meta` (tamper-evident). The runner reads the interpretation from there
  (see [Running the runner](/ops/runner)).
</ParamField>

<ParamField path="deadline" type="timestamp">
  The oracle's resolution deadline. Encoded as an `i64` unix timestamp in the
  instruction payload.
</ParamField>

<ParamField path="KASS / USDC mints" type="address">
  The staking and quote mints, defaulted from the Protocol singleton. KASS is the
  bond/stake asset; USDC is the challenge-market quote asset.
</ParamField>

The TWAP window (`twap_window`) — the length of the challenge-market TWAP window, in seconds — is also part of the create payload (`sdk/src/instructions/lifecycle.ts` `CreateOracleArgs`).

## The instruction it builds

`createOracle` produces the `create_oracle` instruction. Its 25-byte payload is (`sdk/src/instructions/lifecycle.ts`):

```
nonce u64 ++ options_count u8 ++ deadline i64 ++ twap_window i64
```

The metadata (subject + option labels + `uri`/`uri_hash`) is written by a second instruction, `write_oracle_meta` (Ix = 23), appended to the **same transaction** so an oracle is never created without its metadata. It fills the companion `oracle_meta` PDA (`[b"oracle_meta", oracle]`).

The **nonce** is the oracle's identity: the Oracle PDA is derived as `[b"oracle", nonce_u64_le]` (`sdk/src/pda.ts` `oracle`). Because the nonce is not stored on-chain but is needed later by finalize-facts, finalize-oracle, and the claim/close/sweep instructions, the app persists it per-browser at create time (`app/src/lib/nonceStore.ts`).

<Note>
Every staking action across the lifecycle requires **KASS** and is denominated in
**raw base units**. A missing KASS associated-token account is created idempotently
on the first action.
</Note>
