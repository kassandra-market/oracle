- **Oracle nonce (DECISION, not a blocker):** `finalizeFacts`/`finalizeOracle` need the oracle `nonce`, which is NOT stored on the Oracle account and cannot be sourced from the read layer. Added `resolveOracleNonce(oraclePubkey, {maxNonce=4096})` — a PURE (no-RPC) scan that re-derives `[b"oracle", nonce_le]` until it matches. Builders accept an explicit `oracleNonce` (tests/E2E pass it; the UI resolves via the scan; oracles created past the scan bound must pass it explicitly).
- **Offline unit tests** (`app/test/finalize.unit.test.ts`, 13): byte-match each builder vs the SDK, assert the tail is threaded, assert `needsAlt` flips exactly at `MAX_LEGACY_TAIL`, the nonce scan recovers/throws, empty-tail validation.
