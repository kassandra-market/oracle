Stack: `@solana/web3.js@3.0.0-rc.2` (the legacy-compat class API — `Address`, `TransactionInstruction`, async `Address.findProgramAddress` — built on `@solana/kit ^6.10.0`), plain `tsc` build, `vitest` + `litesvm@1.2.0` for offline tests, surfpool gated behind an env flag for e2e. The Rust `sdk-rs/src/{pda,ix,metadao}.rs` is the wire-format source of truth; the TS SDK mirrors it and a parity test pins the shared values.

**Decisions (confirm if reversing):** full SDK; Jupiter = thin offline-testable helper (app wires the live HTTP API); minimal in-SDK Kassandra-oracle reader (phase/resolved_option/options_count) rather than a cross-repo `@kassandra/sdk` dependency.

## Task 5: High-level flows + Jupiter helper + README

**Files:** `sdk/src/flows/{compose,trade,redeem}.ts`, `sdk/src/flows/jupiter.ts`, `sdk/src/flows/index.ts`; `sdk/README.md`; barrel `sdk/src/index.ts`.

**Steps:**
1. `flows/compose.ts`: `composeMarketInstructions(market, oracle, kassMint, payer)` → the ordered `initializeQuestion + initializeConditionalVault + createAmm` instruction list (+ derived addresses) that a client sends before `activate`. Mirrors the Rust harness `compose_metadao_market`.
2. `flows/trade.ts`: `buy(outcome, kassAmount)` = `splitTokens` + a `swap` of the unwanted leg → net YES/NO position; `sell(...)` = mirror; return the instruction list + expected addresses. `redeem.ts`: `redeem(user, marketRefs)` building the vault `redeemTokens` ix. These wrap the raw MetaDAO builders so app code never touches conditional tokens directly.
3. `flows/jupiter.ts`: a THIN, offline-testable helper — `buildJupiterEntryRequest(inputMint, kassMint, amount, ...)` returns the typed Jupiter quote/swap request params + a `composeWithEntry(jupiterSwapIx, marketIx[])` combiner. The actual `fetch` to `quote-api.jup.ag` lives in the app; the SDK only shapes the request and stitches the returned swap instruction in front of the market instructions. Document this boundary clearly. No network in tests.
4. `README.md`: quick-start (install, build, the buy/sell/redeem flow, the compose→activate keeper flow, the Jupiter boundary). Barrel `index.ts` re-exports everything with `metadao.*` and `flows.*` namespaces.
5. `pnpm --filter @kassandra-market/sdk typecheck && test` green. Two-stage review of the wire-format-critical pieces (builders + decoders + MetaDAO parity). Update the design-doc status. Commit `feat(sdk): high-level flows + Jupiter entry helper + README`.
